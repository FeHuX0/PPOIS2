# POIS1: Обслуживание банковской карты (CLI)

## Индивидуальный вариант и предметная область
Реализована программная система для предметной области: использование и обслуживание банковской карты.

Ключевые сущности:
- банковская карта
- пин-код
- банк
- владелец карты
- транзакции
- лимиты

Поддерживаемые операции:
- снятие денег
- внесение денег
- оплата товаров и услуг
- проверка баланса
- блокировка и разблокировка карты
- изменение лимитов

## Объектно-ориентированный анализ
### Сущности
- `Bank`: данные банка (`name`, `bic`).
- `CardOwner`: данные владельца карты (`full_name`, `owner_id`).
- `CardLimits`: лимиты операций (`withdrawal_limit`, `payment_limit`).
- `Transaction`: запись о выполненной операции.
- `BankCard`: агрегат карты (номер, PIN, статус, баланс, лимиты, журнал транзакций).

### Варианты использования (use cases)
- пользователь авторизуется PIN-кодом
- пользователь выполняет финансовую операцию
- система проверяет статус карты, лимиты, баланс
- система обновляет состояние карты и журнал транзакций

### Разделение ответственности (SoC, SOLID)
- `src/domain`: только доменные модели и инварианты.
- `src/services/interfaces.py`: абстракция операций (`CardServiceInterface`).
- `src/services/card_service.py`: бизнес-правила и оркестрация сценариев.
- `src/cli/app_cli.py`: только ввод/вывод и маршрутизация команд CLI.
- `src/core`: перечисления и исключения.

### Enum и Typing
- Состояния карты описаны через `CardStatus` (`Enum`).
- Состояние CLI и команды меню описаны через `CliState` и `CliCommand` (`Enum`).
- Используются стандартные аннотации типов Python (`str`, `Decimal`) без отдельного модуля type aliases.
- Создание демонстрационных объектов вынесено в фабрику `src/factories/card_factory.py`.
- Все публичные методы и ключевые внутренние методы имеют аннотации типов.

## Запуск CLI
```bash
python main.py
```

Демо-параметры карты:
- PIN: `1234`
- стартовый баланс: `10000.00`
- лимит снятия: `1000.00`
- лимит оплаты: `1500.00`

## Запуск unit-тестов
```bash
python -m unittest
```

## Обработка ошибок
Используется иерархия пользовательских исключений:
- `InvalidPinError`
- `CardBlockedError`
- `CardAlreadyBlockedError`
- `CardAlreadyActiveError`
- `InsufficientFundsError`
- `LimitExceededError`
- `InvalidAmountError`

Все ошибки корректно перехватываются в CLI и выводятся пользователю в понятном виде.
